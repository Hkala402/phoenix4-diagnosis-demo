About this project

Tavus Phoenix-4 is an AI replica generator. When a customer's training video fails validation, they get a generic rejection with no explanation of what went wrong. This project is a diagnostic system that runs 8 independent checks and returns a complete list of unmet requirements so the customer can fix everything in one resubmission.

How it works

  1. The customer's video enters the pipeline
  2. 8 independent detectors run — each one checks one specific requirement
  3. Detectors never short-circuit. Every issue is collected, not just the first one
  4. Findings are assembled into a structured report with reason codes, customer-facing messages, and concrete fix actions

The 8 checks

1 Consent statement detection
2 Talking segment duration
3 Silent segment analysis
4 File size validation
5 Video codec check
6 Source type (real human vs AI)
7 Face & lip checks
8 Resolution & aspect ratio

Architecture notes

This page is the live demo. It runs the technical checks (size, codec hint, resolution, aspect ratio, duration) directly on the uploaded file using the browser's video API. The ML-based checks (consent transcription, voice activity, deepfake detection, face/lip sync) are simulated in the demo through the scenario buttons — in the production Python backend, those slots are wired to real models like Whisper, silero-vad, and SyncNet.

The full Python backend with FastAPI service, pytest suite, and pluggable detector interface is in a separate repository.
